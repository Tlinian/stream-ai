## 2. 核心流程

### 2.1 用户输入流程 (输入提示词 → 生成/优化规则)

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户输入   │────▶│  LLM 理解   │────▶│  规则生成   │────▶│  规则引擎   │
│  提示词    │     │   意图      │     │  /优化规则   │     │   更新规则   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 2.2 主数据流转 (CDC数据 → 规则拦截 → 分析 → 报告)

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  源数据库    │────▶│   YDS CDC   │────▶│   规则引擎   │────▶│  数据流处理  │────▶│  报告生成    │
│             │     │   (实时捕获) │     │   (实时拦截) │     │  (清洗/压缩) │     │  (按需)     │
│  毫秒级捕获   │     │  秒级同步    │     │  毫秒级      │     │  毫秒级      │     │             │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
        │                                   │
        │                                   ▼
        │                          ┌─────────────┐
        │                          │  非拦截数据  │────▶ LLM定期分析 ──▶ 新规则/优化
        │                          │  (进入L3)   │
        │                          └─────────────┘
        │
        └────────────────────────▶  报告生成 ◀── LLM分析结果
```

### 2.3 规则优化流程 (报告异常 → 用户输入 → 优化规则)

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  报告中的   │     │  用户输入   │     │  LLM        │
│  异常情况   │────▶│  原始想法   │────▶│  优化规则   │
│             │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                              ▼
                                    ┌─────────────────────┐
                                    │  更新规则引擎        │
                                    │  (越用越聪明)        │
                                    └─────────────────────┘
```

## 6. 核心特性

| 特性 | 描述 |
|------|------|
| **实时性** | 端到端秒级同步，源端毫秒级捕获，规则毫秒级拦截 |
| **智能化** | LLM 理解用户意图，自动生成和优化规则 |
| **三层决策** | 规则引擎(实时) → 数据流处理 → LLM定期分析(非实时) |
| **自进化** | 规则引擎根据异常情况自动优化，越用越聪明 |
| **按需交付** | 报告生成和邮件通知按用户需求配置 |
